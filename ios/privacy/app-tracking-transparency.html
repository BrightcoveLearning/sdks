Introduction

With iOS 14.5, there is a new privacy feature called App Tracking Transparency. You can use this feature in your apps to request tracking permission from your users. When users say no to tracking, the app will not be able to use Apple's ID for Advertisers (IDFA) or other identifiers, like hashed email addresses, to share data about your activity for ad-targeting.

If your app collects end-user data and shares it with other companies for tracking purposes across apps and web site, then you must use the AppTrackingTransparency framework.

For details, see the following:

  • App Store: User Privacy and Data Use
  • App Tracking Transparency

Implementation

To implement App Tracking Transparency, follow these steps:

  1. Review the BasicIMAPlayer sample. This shows how to display the App Tracking Transparency alert to determine if we can use the IDFA or not.
  2. Open the BaseViewController, and locate the viewDidLoad method.
  3. For iOS 14, the SDK checks the value of ATTrackingManager.trackingAuthorizationStatus. If the value is ATTrackingManagerAuthorizationStatusAuthorized, then the SDK uses the IDFA.
  4. For older iOS versions, the SDK uses the value of ASIdentifierManager.sharedManager.isAdvertisingTrackingEnabled to determine if the IDFA should be used.
