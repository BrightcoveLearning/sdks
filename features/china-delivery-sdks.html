Introduction

Delivering video to China from outside the country is challenging due to regulations and controls, restricting the free flow of data across its national borders.

Historically, Brightcove has provided a solution that allowed its services to distribute video content through a global delivery network. However, essential communications, including Playback API requests and responses, and Analytics beacons, could be unreliable, and at times non-functional, when delivering content into areas with stricter controls on information flow.

Brightcoveâ€™s China Delivery solution addresses the problems listed above and comprises two parts:

  • A proxy server hosted within China
  • A connected China-specific CDN

The proxy routes non-video traffic (Playback API requests, Analytics beacons, and the web player itself) to and from viewers in China to Brightcove infrastructure located outside.

The CDN is connected to the proxy and handles the video and image content.

Requirements

The following requirements are needed for China Delivery with the Brightcove Native SDKs:

Device OS version

  • Android 6.0 and newer

    Android 8.x may have issues with offline rental licenses.
  • iOS 11.0 and newer

Brightcove SDK version

  • Native SDK for Android (coming soon)
  • Native SDK for iOS 6.5.0 and newer

Platform

  • A Brightcove Enterprise account - for details, contact your account manager
  • A Video Cloud account enabled for China Delivery
    • This account can deliver inside and outside of China; a separate account is optional, but not required
  • A valid Internet Content Provider (ICP) license
  • An approved domain or subdomain name listed on the ICP license

Enable China Delivery

China Delivery is a paid add-on for your account. Contact your account manager to enable this feature.

Android Implementation

Contact your account manager to get a pre-release of the Native SDK for Android that supports China Delivery.

iOS Implementation

To use China Delivery with the Native SDK for iOS, follow these steps

  1. The BCOVGlobalConfiguration class allows you to set custom configurations for the Native SDK for iOS.

  2. Within this class, set the chinaProxyDomain property to a custom, fully qualified domain name (FQDN) for the Playback API, metrics and analytics collection services.

    @property (nonatomic, nullable) NSString *chinaProxyDomain

Notes

The following features are NOT supported when using China Delivery with the Native SDKs:

  • Live streaming
  • Dynamic Delivery Rules
  • Static URLs for Dynamic Delivery
  • Integrations requiring 3rd-party server access are not supported, including:
    • DRM
    • Advertising (CSAI + SSAI)
